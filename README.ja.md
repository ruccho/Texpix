# Texpix

[English](README.md)

<img width="720" alt="Image" src="https://github.com/user-attachments/assets/136cda22-0931-4141-a566-9528d3ca30ca" />

**Unity (uGUI) 向けのピクセルフォント描画システムです。**

Texpix はピクセルフォントを常に鮮明に表示します。フォントの1ピクセルが画面上の1ブロックになり、ぼやけたり、中途半端に欠けたりしません。TextMesh Pro を無理やり設定で押さえ込むのではなく、自然で一貫したやり方でピクセルフォントを表示します。

## なぜ TextMesh Pro ではなく Texpix なのか

TextMesh Pro は SDF (signed distance field) を軸に設計されており、どんなサイズでも滑らかに見えることを目指しています。しかし、ピクセルフォントではむしろその逆が求められます。

- **常にピクセルパーフェクト：** SDF, アンチエイリアス、サンプリング設定が不要です。グリフはフォント本来のピクセルサイズで一度だけラスタライズされ、ピクセル補間なしで、ピクセルグリッドにスナップして描画されます。フォントのピクセルサイズと整数スケールを指定するだけで、どんな拡大率でも正しく表示されます。
- **実用的なアウトライン：** ゲームUIでよく利用される1pxアウトライン（4近傍・8近傍）が設定1つで有効になります。TMP のアウトラインは距離フィールドの dilate によるもので、きれいな1px近傍アウトラインを作るのが大変でした。Texpix ではアウトライン情報がアトラスに埋め込まれているため、最低限のテクスチャサンプリングと最低限の描画パスでアウトラインを描画します。
- **省サイズのアトラス：** グリフの各ピクセルは4状態しか必要としないため、R8 テクスチャの1バイトにフォントピクセル4つを詰め込んでいます。同等の RGBA アトラスと比べておおよそ 1/8 のメモリで、しかもアウトライン情報まで含めることができます。

## 動作要件

- Unity 6000.3 以降
- uGUI (Canvas)。UI Toolkit には対応していません。

## インストール

Package Manager → **Add package from git URL…**:

```
https://github.com/ruccho/Texpix.git?path=Packages/com.ruccho.texpix#release
```

## クイックスタート

1. ピクセルフォントの TTF をプロジェクトにインポートします。
2. **Assets → Create → Texpix → Font Asset** を作成し、**Source Font** に TTF を割り当て、**Pixel Size** にそのフォント本来のサイズを設定します。ここが最大のポイントで、この値がずれていることがピクセルフォントの表示がぼやける原因です。

<img width="500" alt="Image" src="https://github.com/user-attachments/assets/4d1152d3-8541-474d-a768-f4071525969f" />

3. Canvas の下に **GameObject → UI → Texpix Text** を作成します。
4. フォントアセットを割り当ててテキストを入力し、**Pixel Scale** にフォント1ピクセルが占めるキャンバス単位数を設定します。整数（1, 2, 3, …）にしてください。

アトラスは必要に応じて動的に構築されるため、制作中にベイクする必要はありません。

## 機能と設定項目

### Texpix Text (コンポーネント)

| 設定 | 内容 |
| --- | --- |
| **Font** | 描画に使う `TexpixFontAsset`。 |
| **Text** | 表示する文字列。`\n` に対応。 |
| **Pixel Scale** | フォント1ピクセルあたりのキャンバス単位数。整数にするとピクセルが正方形のまま鮮明に保たれます。 |
| **Horizontal / Vertical Alignment** | Left / Center / Right × Top / Middle / Bottom の9通り。 |
| **Wrap Mode** | `NoWrap`、または `Wrap`（スペース区切りテキストはワードラップ、CJK は文字単位で簡易禁則処理つき）。 |
| **Overflow** | `Overflow`（rect の外にもそのまま描画）、`Truncate`（入りきらない分を捨てる）、`Ellipsis`（切り詰めて `…` を付加）。 |
| **Letter Spacing** | 文字・スプライト間の追加スペース（フォントピクセル単位、負値可）。カーニングと同様に折り返しや整列にも反映されます。 |
| **Line Spacing** | フォントの行高に加算される値（フォントピクセル単位、負値可）。 |
| **Snap To Pixel Grid** | レイアウト原点を Pixel Scale の倍数に丸め、グリフのテクセルが 1:1 に保たれるようにします。基本的にオンのままで。 |
| **Rich Text** | 下記のタグセットを有効にします。 |
| **Sprite Asset** | `<sprite>` タグが参照する `TexpixSpriteAsset`。 |
| **Outline Mode** | `None` / `FourNeighbor` / `EightNeighbor`。 |
| **Outline Color** | アウトラインピクセルの色。 |

本コンポーネントは `MaskableGraphic` なので、`Mask` と `RectMask2D` は通常どおり機能します。

### リッチテキスト

意図的に絞り込んだタグセットです。

- `<color=#RRGGBB>` / `<color=red>` … `</color>`（ネスト可）
- `<sprite=name>` / `<sprite index=0>`、`tint=1` で現在の色に着色
- `<noparse>…</noparse>`
- `<br>`

未知のタグは、握りつぶさずにそのまま文字として描画されます。

### Font Asset

| 設定 | 内容 |
| --- | --- |
| **Source Font** | ラスタライズ元の TTF/OTF。 |
| **Pixel Size** | フォント本来のデザインサイズ（ピクセル）。 |
| **Atlas Mode** | `Dynamic`（必要に応じてグリフを追加）または `Static`（エディタでベイク済み）。 |
| **Fallback Fonts** | このフォントに無いグリフを順に探す別のフォントアセット。チェーンは深さ優先で平坦化され、循環は無視されます。 |
| **Atlas Width / Max Height** | 動的アトラスのサイズ上限。小さく始まり、必要に応じて高さが倍々に拡張されます。 |
| **Bake Character Set** | **Bake** で静的アトラスに書き込む文字。 |

インスペクタには **Bake** / **Clear** ボタンとアトラスのプレビューがあります。ベイクするとグリフテーブル・カーニングペア・メトリクス・アトラステクスチャがサブアセットとして直列化され、アセットが Static に切り替わります。

### Sprite Asset

**Assets → Create → Texpix → Sprite Asset**: RGBA テクスチャと、名前付きエントリ（ピクセル矩形、ベアリング、アドバンス）で構成されます。テクスチャを Sprite Mode *Multiple* でインポートしていれば、**Import Entries From Texture Sprites** でテーブルを自動生成できます。インラインスプライトはベースライン上に配置され、レイアウトや折り返しにも参加します。

### カスタムシェーダ

`Runtime/Shaders/Texpix.hlsl` は公開インクルードで、アトラスのデコードAPI (`TexpixAtlasUV`, `TexpixSubPixel`, `TexpixExtractLevel`, `TexpixShade`) を提供します。Package Manager から **Custom Shader** サンプルをインポートすると、動作する実例（UI の stencil/clip 定型処理を含む、色相が循環する塗り）を確認できます。作成したマテリアルはコンポーネント標準の `material` スロットに割り当ててください。フォールバック用のサブレンダラにも引き継がれます。

# 上級者向け

## 動作原理

**アトラスはフォントピクセルあたり2ビット：** グリフをラスタライズする際、各ピクセルは4つのレベル、すなわち *塗り* / *4近傍アウトライン* / *斜めのみのアウトライン* / *外側* のいずれかに分類されます。この分類はビットマップの周囲1ピクセルの余白リングを使って、取り込み時に一度だけ計算されます。この4ピクセルが R8 テクスチャの1バイトにパックされます。

**アウトラインはシェーダのスイッチ：** アウトラインの2種類はすでにアトラスに入っているため、`Outline Mode` はシェーダがどのレベルを不透明として扱うかを変えるだけです。`None` はレベル3のみ、`FourNeighbor` はレベル2を追加、`EightNeighbor` はさらにレベル1を追加します。2パス目も dilate もテクスチャ追加もありません。

**頂点はアトラス上のピクセル座標を保持する：** クアッドの UV は正規化 UV ではなく、アトラス上の整数フォントピクセル座標です。テクセル内のどのサブピクセルかはシェーダ側でデコードします。レイアウトも同様に整数フォントピクセル空間で完結し、メッシュ構築時に Pixel Scale で拡大されます。結果が「だいたい揃っている」ではなく厳密に一致するのはこのためです。

**グリフは Unity の FontEngine から取得：** 動的アセットは必要になった時点でラスタライズ（RASTER モード、閾値処理で2値化）し、固定セルグリッドに O(1) でパックします。静的アセットはベイク済みのグリフテーブルとアトラスを持ち、実行時に FontEngine を一切使いません。

## 性能を出すための原則

- **リリース時は静的アトラスをベイクする：** Static モードは実行時のラスタライズ、FontEngine への依存、新しいテキストが出現したときの初回フレームのひっかかりをなくします。動的モードは制作中向け、静的モードはリリース向けです。ただし表示内容が本質的に無限（ユーザー名、チャット、事前に分からないローカライズテキスト）である場合は例外です。
- **`Atlas Max Height` は実際に使うグリフ集合に合わせる：** 動的アトラスが満杯になると、グリフ単位で追い出すのではなくアトラス全体をクリアして再構築します。安価で予測しやすい代わりに、1フレームだけ表示が乱れ、警告がログに出ます。この警告が繰り返し出る場合、同時に画面に出るグリフ数に対してアトラスが小さすぎます。
- **フォールバックチェーンは短く：** テキストに実際にグリフを供給したフォールバックフォントごとに、専用のアトラステクスチャを持つ子レンダラが追加されます。つまりテキスト1つにつき、フォールバック1つあたり描画コールが1つ増えます。よく使う文字は主フォント側に入れてください。インラインスプライトも同様で、スプライトテクスチャ用に子レンダラが1つ増えます。
- **マテリアル共有でバッチングを効かせる：** 既定では `Texpix Text` はコンポーネントごとにマテリアルを実体化します（コンポーネント単位のアウトライン色・モードをここに保持するため）。そのためコンポーネント同士はバッチされません。同じフォントとアウトライン設定のラベルが画面に多数並ぶ場合は、`Texpix/UI Default` からマテリアルを1つ作り、`_OutlineMode` と `_OutlineColor` を設定して、各コンポーネントの `material` スロットに割り当ててください。この場合、コンポーネント側の Outline Mode / Outline Color は反映されず、マテリアルの値が優先されます。
- **Pixel Scale は整数にし、キャンバス自体をピクセルパーフェクトに保つ：** 小数のスケール、整数でないキャンバススケールファクタ、回転・拡縮された親があると、スナップでは打ち消せないサンプリング誤差が再び発生します。
- **`Text` の変更はレイアウトの再実行を伴う：** レイアウト処理は整数演算のみでアロケーションも少ないものの、文字数に対して O(n) です。長い文字列を毎フレーム書き換えるのは避け、頻繁に変わる部分（カウンタやタイマー）は小さな別コンポーネントに分けてください。
- **アトラスの拡張は遅延するが無料ではない：** 動的アトラスの高さが倍になるとテクスチャが再生成され、それを使う全テキストが次のキャンバス更新で再構築されます（1フレーム遅れ）。頻度は低く自動的に収束しますが、これもリリース前にベイクしておくべき理由の一つです。

## 制限事項

- uGUI 専用です。UI Toolkit 非対応、TextMeshPro のドロップイン置き換えでもありません。
- 両端揃え、行高の倍率指定には未対応です。
- テキストコンポーネント1つにつきスプライトアセットは1つ。スプライトのクアッドは常にグリフより手前に描画されます。
- カーニングはフォントフォールバックの境界をまたいでは適用されません。
- 静的アトラスは内容の高さに合わせてトリミングされません。
